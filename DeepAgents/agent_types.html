01

Shallow Agent

The simplest form of an agent. Receives input, makes a single LLM call, optionally calls one tool, and returns the output. No loop. No planning. No self-correction.

When to use

  • Answering a simple factual question
  • Summarizing a document in one pass
  • Classifying or labeling data
  • Single-step extraction tasks

Limitations

  • Cannot handle multi-step problems
  • No recovery from errors or bad results
  • No memory between steps
  • One shot — cannot retry or refine

02

ReAct Agent

Short for Reasoning + Acting. The agent loops between thinking about what to do, calling a tool, and reading the result — repeating until the task is complete. The workhorse of modern AI agents.

When to use

  • Tasks requiring multiple tool calls in sequence
  • Q&A over external data sources
  • Code generation with self-debugging
  • Any task needing verification or refinement

Limitations

  • Can get stuck in infinite loops
  • More loops means higher latency and cost
  • Context window grows with every step
  • Not designed for parallel sub-tasks

03

Deep Agent

A multi-layered system built for complex, long-horizon goals. A central Orchestrator coordinates multiple specialist sub-agents running in parallel. An Evaluator checks quality before responding, and a Memory Store persists knowledge across sessions.

When to use

  • Research spanning many sources and steps
  • End-to-end software engineering pipelines
  • Complex data analysis and reporting
  • Tasks that need a team of human specialists

Limitations

  • High latency — many agent round-trips
  • High cost — far more LLM calls per task
  • Harder to debug when failures go deep
  • Requires careful orchestration design

01 / Shallow

Shallow Agent

Use when the task is simple and predictable. Single LLM call, no loop, no memory. Fast and cheap.

02 / ReAct

ReAct Agent

Use when the task requires multiple tool calls, iteration, and self-correction. The workhorse of modern AI agent systems.

03 / Deep

Deep Agent

Use when the task is complex enough to need a team — parallel execution, long-term memory, orchestration, and evaluation.
